# E2E tests

Headless-Chromium end-to-end tests via Playwright. They exercise the running
dev stack — no mocks.

## Prerequisites

1. Dev stack up: `./dev.sh` from repo root (backend on :8000, frontend on :3000).
2. Demo user present (auto-provisioned on backend startup).
3. Playwright dependencies installed: `npm install` in `frontend/`.
4. Chromium binary on disk. Already cached at
   `~/.cache/ms-playwright/chromium-1208/chrome-linux64/chrome`. If missing,
   run `npx playwright install chromium` once.

## Run

```bash
cd frontend
npm run test:e2e              # headless
npm run test:e2e:headed       # see the browser
npm run test:e2e:ui           # Playwright UI mode for picking selectors
npx playwright test upload-live --headed   # run a single spec
```

## Files

- `playwright.config.ts` — base URL, local chromium path, retries.
- `auth.setup.ts` — logs in as the demo user once, stores session in
  `.auth/user.json`. Every spec reuses that session.
- `fixtures.ts` — `getOrCreateProjectId()` helper that talks to the backend
  REST API to ensure at least one project exists.
- `upload-live.spec.ts` — drops a file and asserts the card progresses through
  pipeline stages and reaches "fertig" *without a page refresh* (the original
  bug regression test).
- `upload-errors.spec.ts` — `.exe` upload → 415 toast; oversized file → 413
  toast.
- `delete-undo.spec.ts` — delete card → vanish → Rückgängig → restore.

## Customising

- `E2E_BASE_URL` — defaults to `http://localhost:3000`.
- `E2E_BACKEND_URL` — defaults to `http://localhost:8000`.
- `E2E_USER_EMAIL` / `E2E_USER_PASSWORD` — defaults to the demo user.
- `E2E_CHROMIUM_PATH` — override if the cached chromium lives elsewhere.

## Auth-state cache

`.auth/user.json` is regenerated by the `setup` project before every test
run, so it's safe to gitignore.
